Introducción a IoT

¿Que significan las siglas IoT?

El Internet de las cosas (IoT, por sus siglas en inglés, Internet of Things) se refiere a la conexión de dispositivos físicos a través de internet, permitiéndoles recopilar y compartir datos. Estos dispositivos pueden ser objetos cotidianos, como electrodomésticos, vehículos, dispositivos de salud, sistemas de gestión de energía, entre otros. La idea central es que estos dispositivos pueden recopilar y intercambiar información de manera autónoma, lo que facilita la automatización, la toma de decisiones inteligentes y la mejora de la eficiencia en diversas aplicaciones.

En un contexto de IoT, los dispositivos están equipados con sensores, actuadores y conectividad a internet, lo que les permite recopilar datos del entorno y realizar acciones en función de esa información. Estos datos pueden ser recopilados, analizados y utilizados para tomar decisiones informadas, optimizar procesos y mejorar la interacción entre humanos y máquinas.

El IoT tiene aplicaciones en una variedad de sectores, como la salud, la agricultura, la industria, el transporte, la domótica y muchos otros. La creciente interconexión de dispositivos a través de IoT está transformando la forma en que vivimos y trabajamos, brindando nuevas oportunidades y desafíos en términos de seguridad, privacidad y gestión de grandes volúmenes de datos.

Historia y evolución del IoT

La historia y la evolución del Internet de las cosas (IoT) se remontan a varias décadas, y su desarrollo ha estado influenciado por avances tecnológicos en diversas áreas. A continuación, se proporciona un resumen de los hitos más destacados en la historia del IoT:

  1. Década de 1970:
    • Los primeros indicios de dispositivos conectados surgieron con la invención de las primeras máquinas expendedoras conectadas a la red para monitorear los niveles de inventario y enviar alertas.
  1. Década de 1980:
    • Surgieron sistemas de identificación por radiofrecuencia (RFID) para rastrear y gestionar inventarios.
  1. Década de 1990:
    • El término "Internet de las cosas" fue acuñado por Kevin Ashton en 1999 mientras trabajaba en Procter & Gamble. Se refería a la idea de etiquetar y rastrear productos a través de la web.
    • Avances en tecnologías de redes y protocolos de comunicación contribuyeron al crecimiento potencial del IoT.
  1. Principios de los años 2000:
    • Se desarrollaron estándares de comunicación, que facilitaron la conexión de dispositivos a la red.
    • La miniaturización de sensores y la reducción de costos permitieron la proliferación de dispositivos conectados.
  1. Mediados de los años 2000:
    • Se produjo la popularización de las plataformas de desarrollo de hardware, como Arduino y Raspberry Pi, que facilitaron la creación de prototipos de dispositivos IoT.
  1. 2010 en adelante:
    • El IoT experimentó un rápido crecimiento con el aumento de la conectividad a través de la banda ancha y la adopción generalizada de dispositivos móviles.
    • Grandes empresas de tecnología, como IBM, Microsoft, y Google, invirtieron en soluciones y plataformas de IoT.
    • Surgieron estándares de IoT, como MQTT, para facilitar la comunicación entre dispositivos.
  1. Actualidad:
    • El IoT se ha expandido a una amplia gama de sectores, incluidos la salud, la agricultura, la industria, el transporte y la domótica.
    • La integración de la inteligencia artificial y el machine learning ha mejorado la capacidad de los dispositivos IoT para procesar y analizar datos.
    • La seguridad y la privacidad en el IoT son áreas de creciente preocupación, y se están desarrollando medidas para abordar estos desafíos.

Aplicaciones IoT

El Internet de las cosas tiene aplicaciones en una amplia variedad de sectores, mejorando la eficiencia, la comodidad y la toma de decisiones en diversos contextos.

Domótica (Smart Homes):

  • Control de luces, termostatos, cámaras de seguridad y electrodomésticos a través de dispositivos conectados.
  • Sistemas de seguridad inteligente que pueden enviar alertas y permitir la monitorización remota.

Salud Conectada:

  • Dispositivos de monitoreo de salud, como pulseras y relojes inteligentes, que registran la actividad física, la frecuencia cardíaca y otros parámetros.
  • Equipos médicos conectados para monitoreo remoto y gestión de enfermedades crónicas.

Agricultura Inteligente (AgTech):

  • Sensores en el campo que recopilan datos sobre suelo, clima y cultivos.
  • Sistemas de riego automatizado basados en la información recopilada por sensores.

Ciudades Inteligentes:

  • Gestión del tráfico mediante sensores y cámaras.
  • Recolección de datos para la planificación urbana, como la monitorización de la calidad del aire y la gestión de residuos.

Industria 4.0:

  • Monitorización y mantenimiento predictivo de maquinaria industrial.
  • Logística y gestión de la cadena de suministro mediante sensores y seguimiento en tiempo real.

Vehículos Conectados:

  • Sistemas de navegación inteligente y asistentes de conducción.
  • Monitoreo remoto de vehículos y gestión de flotas.

Energía Inteligente:

  • Medidores inteligentes para monitorizar y gestionar el consumo de energía.
  • Sistemas de gestión de energía en el hogar y en entornos empresariales.

Retail (Comercio Minorista) Inteligente:

  • Etiquetas RFID para el seguimiento de inventario.

Wearables y Fitness:

  • Dispositivos de fitness conectados que monitorizan la actividad física y la salud.
  • Integración de datos de wearables con aplicaciones de salud y bienestar.

Gestión Ambiental:

  • Sensores para el monitoreo de la calidad del aire y del agua.
  • Sistemas de alerta temprana para eventos climáticos extremos.

Sensores y actuadores

La base del Internet de las Cosas

El Internet de las Cosas ha emergido como una revolución tecnológica que conecta el mundo físico con el virtual, permitiendo la interacción entre dispositivos y sistemas de manera más inteligente y eficiente. En el corazón de esta transformación se encuentran los sensores y actuadores, componentes esenciales que habilitan la recopilación de datos y la ejecución de acciones en el entorno físico.

Los sensores son dispositivos que detectan y convierten información física del entorno en señales eléctricas. Su diversidad abarca desde simples termómetros hasta complejos sensores de imagen y sensores biométricos. La clave de su importancia radica en su capacidad para proporcionar datos en tiempo real sobre variables como temperatura, humedad, luz, movimiento y más. En el contexto del IoT, los sensores sirven como los ojos y oídos digitales del mundo físico, permitiendo la monitorización continua de variables críticas.

Por otro lado, los actuadores son dispositivos que ejecutan acciones en respuesta a señales eléctricas o comandos. Pueden variar desde motores y válvulas hasta dispositivos más sofisticados como brazos robóticos. La función fundamental de los actuadores en el contexto del IoT es transformar datos y decisiones digitales en cambios físicos en el entorno. Por ejemplo, un actuador en un sistema de domótica puede cerrar cortinas automáticamente en respuesta a la información de un sensor de luz ambiental.

La convergencia efectiva de sensores y actuadores en el IoT permite la creación de sistemas autónomos y reactivos. En la gestión del tráfico urbano, los sensores monitorean el flujo vehicular, y los actuadores controlan las luces del semáforo para optimizar el flujo. En la fabricación, sensores detectan anomalías en el proceso, y los actuadores realizan ajustes instantáneos para mantener la calidad del producto.

A pesar de los beneficios evidentes, el despliegue masivo de sensores y actuadores también plantea desafíos, especialmente en áreas de seguridad y privacidad. La recopilación y transmisión de datos constantes requiere una atención cuidadosa para garantizar la protección de la información sensible y prevenir posibles vulnerabilidades.

Placas de desarrollo

Una placa de desarrollo es una plataforma de hardware que incluye componentes esenciales, como un procesador, memoria, puertos de entrada/salida y, a menudo, conectividad a redes, todo empaquetado en una forma compacta y fácil de usar. Uno de los ejemplos más emblemáticos es Arduino, que ha cautivado a una audiencia global con su simplicidad y versatilidad. Raspberry Pi, otra figura destacada, ha llevado la potencia de cómputo completa a dimensiones reducidas, permitiendo a los usuarios crear desde simples proyectos educativos hasta soluciones de servidores ligeros.

Las placas de desarrollo han alterado el paradigma tradicional del desarrollo de hardware, abriendo las puertas a la creatividad y la experimentación. Su contribución se manifiesta en diversas áreas, desde la educación hasta la investigación y el desarrollo comercial. En entornos educativos, estas plataformas ofrecen una introducción práctica a los principios de la electrónica y la programación, estimulando la curiosidad y fomentando el aprendizaje activo.

En el ámbito de la investigación, las placas de desarrollo sirven como bancos de pruebas ideales para prototipos rápidos y experimentación rápida. Investigadores y desarrolladores pueden llevar a cabo pruebas de concepto sin la necesidad de costosos equipos personalizados, acelerando el ciclo de desarrollo y permitiendo la exploración de nuevas ideas con mayor agilidad.

Desde el punto de vista comercial, el auge de las placas de desarrollo ha facilitado la incubación de nuevas empresas y la materialización de productos innovadores. Empresas emergentes pueden utilizar estas plataformas para crear prototipos de productos y demostrar la viabilidad de sus ideas a inversores y clientes potenciales antes de comprometerse con grandes inversiones en la fabricación.

Un aspecto crucial de las placas de desarrollo es su carácter de código abierto y la rica comunidad que las respalda. Los entusiastas y desarrolladores de todo el mundo comparten conocimientos, códigos y proyectos, creando un ecosistema colaborativo que potencia la creatividad colectiva. Esto no solo impulsa el desarrollo de hardware, sino que también nutre la evolución continua de las tecnologías relacionadas.

Conectividad y tecnología

En la era digital actual, la conectividad se ha convertido en el tejido que une y capacita la vasta red de dispositivos que pueblan nuestro mundo interconectado. Tecnologías como WiFi, Bluetooth y Zigbee, entre otras, desempeñan roles fundamentales al posibilitar la comunicación sin fisuras entre dispositivos, desde los más pequeños y portátiles hasta los sistemas más complejos en la Internet de las Cosas (IoT).

Tecnologías inalámbricas

WiFi, una de las tecnologías más ubicuas, ha transformado la forma en que accedemos y compartimos información. Desde la comodidad de nuestros hogares hasta espacios públicos, la conectividad inalámbrica ha liberado a los dispositivos de las ataduras de los cables, permitiendo una movilidad sin precedentes. La velocidad y la fiabilidad de las redes WiFi han propiciado un aumento exponencial en el intercambio de datos, desde la transmisión de medios hasta la colaboración en tiempo real, facilitando la vida moderna y empoderando nuevas formas de trabajo y comunicación.

Bluetooth, por otro lado, se ha consolidado como la columna vertebral de la conectividad de dispositivos personales. Auriculares inalámbricos, altavoces, teclados y una multitud de dispositivos periféricos utilizan Bluetooth para establecer conexiones confiables y eficientes. La capacidad de emparejamiento sencillo y el bajo consumo de energía han convertido a Bluetooth en un estándar para la comunicación inalámbrica en el ámbito personal.

En el ámbito de la Internet de las Cosas (IoT), donde la conexión de dispositivos se expande más allá de los límites tradicionales, Zigbee ha emergido como una tecnología clave. Diseñada para el bajo consumo de energía y la comunicación eficiente entre dispositivos, Zigbee encuentra aplicaciones en la automatización del hogar, la monitorización industrial y otros escenarios donde la eficiencia energética y la confiabilidad son fundamentales. Su estructura de malla permite la creación de redes robustas y adaptables, lo que es esencial en entornos dinámicos y en constante cambio.

La conectividad, sin embargo, no se limita solo a estas tecnologías. El despliegue de la 5G, la quinta generación de redes móviles, está dando forma a un futuro donde la conectividad ultrarrápida y la baja latencia serán la norma. Esto no solo mejorará la experiencia del usuario en términos de velocidad de descarga, sino que también abrirá las puertas a nuevas aplicaciones, desde la realidad aumentada hasta la conducción autónoma.

El famoso ESP32

La Revolución inalámbrica en desarrollo IoT

En el escenario del desarrollo de la Internet de las Cosas, el ESP32 ha surgido como una potente herramienta, una verdadera joya para los innovadores y desarrolladores que buscan llevar a cabo proyectos inalámbricos sofisticados y eficientes. Desarrollado por Espressif Systems, el ESP32 ha redefinido las posibilidades de conectividad, combinando versatilidad, rendimiento y eficiencia energética en una pequeña placa de desarrollo.

En el corazón del atractivo del ESP32 está su capacidad para proporcionar conectividad WiFi y Bluetooth en un solo chip. Este logro no solo simplifica la integración de estas tecnologías en proyectos, sino que también amplía el alcance de aplicaciones potenciales. Ya sea controlando dispositivos domésticos inteligentes a través de WiFi o estableciendo comunicación inalámbrica entre dispositivos periféricos mediante Bluetooth, el ESP32 se presenta como una solución integral.

La arquitectura del ESP32, con su procesador de doble núcleo y capacidad de procesamiento de señales digitales, ofrece un rendimiento impresionante para una placa de su tamaño. Esto es especialmente valioso en proyectos que requieren cálculos intensivos, como procesamiento de señales, reconocimiento de voz o aplicaciones de aprendizaje automático. La capacidad de ejecutar múltiples tareas simultáneamente sin comprometer el rendimiento ha convertido al ESP32 en una opción preferida para proyectos exigentes.

En el ámbito del bajo consumo de energía, una característica crucial para muchos dispositivos IoT, el ESP32 brilla con su capacidad de operar en modos de bajo consumo sin sacrificar la funcionalidad. El modo de suspensión profunda y las opciones de gestión de energía permiten a los desarrolladores optimizar el consumo de energía según los requisitos específicos de sus proyectos, lo que es vital para aplicaciones alimentadas por batería.

La comunidad activa y colaborativa que rodea al ESP32 también ha contribuido significativamente a su éxito. La documentación exhaustiva, las bibliotecas en constante expansión y la participación activa en foros y comunidades en línea han creado un ecosistema que facilita el aprendizaje y el desarrollo. Esto ha atraído a una gama diversa de usuarios, desde estudiantes y aficionados hasta profesionales de la industria, fomentando la creatividad y la innovación.

Configuración

La placa de desarrollo ESP32 necesita comunicarse con el computador y para lograrlo necesita un intermediario. Este componente es el driver, y debe ser instalado.

Esta placa utiliza el chip CH340.

Instalación del driver CH340

En Windows 10 el driver es automaticamente instalado por el sistema operativo. De no ser así, seguir los siguientes pasos:

  1. Abrir Administrador de dipositivos.
  1. Boton derecho sobre el USB-Serial y clikear en actualizar driver.
  1. Buscar el driver en la carpeta proporcionada.
  1. El driver se instalará y ahora debemos configurar el entorno de programación del software Arduino.

Configuración del IDE Arduino

Para poder programar el ESP32 con este IDE(Integrated Development Enviroment) se debe agregar la tarjeta de desarrollo a la base de datos de tarjetas del programa.

  1. Abrir el IDE de Arduino y dirigirse a Preferencias.
  1. Copiar el siguiente link y pegarlo donde indica la imagen. 
https://dl.espressif.com/dl/package_esp32_index.json
  1. En Herramientas dirigirse a Gestor de tarjetas.
  1. Buscar ESP32 e instalar.
  1. En Herramientas - > Tarjetas buscar ESP32 Arduino y seleccionar ESP32 Dev Module.

Cargando sketchs

void setup(){

}

}

Si este programa funciona entonces estamos listos para comenzar a programar los componentes.

Sketch 3 - Temperatura y humedad ambiente

¿Que es I2C?

Es un protocolo de comunicación serial que permite la transferencia de datos entre dispositivos electrónicos.

Nosotros la utilizaremos para comunicar la pantalla y el ESP32.

#include <Wire.h>
#include <LiquidCrystal_I2C.h>
LiquidCrystal_I2C mylcd(0x27,16,2);
#include "xht11.h"
xht11 xht(17);

unsigned char dht[4] = {0, 0, 0, 0};//Only the first 32 bits of data are received, not the parity bits
void setup() {
  Serial.begin(9600);//Start the serial port monitor and set baud rate to 9600
  mylcd.init();
  mylcd.backlight();
}

void loop() {
  if (xht.receive(dht)) { 
    Serial.print("RH:");
    Serial.print(dht[0]); 
    Serial.print("%  ");
    Serial.print("Temp:");
    Serial.print(dht[2]);
    Serial.println("C");
    
    mylcd.setCursor(0, 0);
    mylcd.print("T = ");
    mylcd.print(dht[2]);
    mylcd.setCursor(0, 1);
    mylcd.print("H = ");
    mylcd.print(dht[0]);
    //mylcd.clear();
    delay(200);
  } else {    //Read error
    Serial.println("sensor error");
  }
  delay(1000);
}

void colorWipe(uint32_t color, int wait) {
  for(int i=0; i<strip.numPixels(); i++) { // For each pixel in strip...
    strip.setPixelColor(i, color);         //  Set pixel's color (in RAM)
    strip.show();                          //  Update strip to match
    delay(wait);                           //  Pause for a moment
  }
}

// Rainbow cycle along whole strip. Pass delay time (in ms) between frames.
void rainbow(int wait) {
  for(long firstPixelHue = 0; firstPixelHue < 5*65536; firstPixelHue += 256) {
    for(int i=0; i<strip.numPixels(); i++) { // For each pixel in strip...
      int pixelHue = firstPixelHue + (i * 65536L / strip.numPixels());
      strip.setPixelColor(i, strip.gamma32(strip.ColorHSV(pixelHue)));
    }
    strip.show(); // Update strip with new contents
    delay(wait);  // Pause for a moment
  }
}

// Rainbow-enhanced theater marquee. Pass delay time (in ms) between frames.
void theaterChaseRainbow(int wait) {
  int firstPixelHue = 0;     // First pixel starts at red (hue 0)
  for(int a=0; a<30; a++) {  // Repeat 30 times...
    for(int b=0; b<3; b++) { //  'b' counts from 0 to 2...
      strip.clear();         //   Set all pixels in RAM to 0 (off)
      // 'c' counts up from 'b' to end of strip in increments of 3...
      for(int c=b; c<strip.numPixels(); c += 3) {
        int      hue   = firstPixelHue + c * 65536L / strip.numPixels();
        uint32_t color = strip.gamma32(strip.ColorHSV(hue)); // hue -> RGB
        strip.setPixelColor(c, color); // Set pixel 'c' to value 'color'
      }
      strip.show();                // Update strip with new contents
      delay(wait);                 // Pause for a moment
      firstPixelHue += 65536 / 90; // One cycle of color wheel over 90 frames
    }
  }
}
